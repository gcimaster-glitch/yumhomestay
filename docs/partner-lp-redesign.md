# パートナーLP全面刷新 設計書

## 1. 問題分析

### 1.1 指摘の本質

複数の方から「35点」と評価された根本原因は、**ビジネスの常識に反する情報設計**です。具体的には以下の3つの構造的欠陥があります。

### 1.2 欠陥①：3パートナーの情報が混在している

現状、ナビゲーションの「パートナー」ドロップダウンから、ホスト・料理教室・旅行代理店の3つのLPと**共通のデモページ**が並列に表示されています。

| 問題箇所 | 現状 | あるべき姿 |
|---|---|---|
| Navbar「パートナー」メニュー | 3LP + デモが同列に並ぶ | 各パートナーが自分のLPだけを見る |
| `/demo` ページ | 3種類のデモが1ページに同居 | 各LP内に独立したデモ入口 |
| デモ後の「詳細はこちら」 | 3つのLP全てへのリンクが表示 | 自分のLPのみ |

ホストが料理教室の情報を見る必要はなく、旅行代理店も他の情報は不要です。むしろ**他パートナーの報酬体系が見えることで、自分のサービスの原価構造が推測できてしまう**という致命的な問題があります。

### 1.3 欠陥②：原価・単価が丸見え

| ページ | 露出している情報 | 問題 |
|---|---|---|
| ForHosts.tsx | 月収¥50,000〜¥400,000、「月2回受入で月収¥50,000以上」 | ホスト報酬の個別単価が推測可能 |
| ForCookingSchools.tsx | 月収¥100,000〜¥400,000 | 料理教室報酬の個別単価が推測可能 |
| ForAgents.tsx | 1件¥2,200の紹介手数料、基本手数料¥8,800 | 代理店手数料の完全な単価表 |
| Pricing.tsx | 基本料金¥55,000、追加大人¥22,000、追加子供¥11,000 | **ゲスト向け料金（＝サービス原価）が完全公開** |
| BusinessHost.tsx | 基本報酬¥20,000、追加大人¥8,000、食材費補助¥5,000 | ホスト報酬の完全な単価表（トークン保護あり） |
| BusinessCookingSchool.tsx | 基本報酬¥20,000、追加大人¥8,000、食材費補助¥5,000 | 料理教室報酬の完全な単価表（トークン保護あり） |
| BusinessAgent.tsx | 基本手数料¥8,800、追加大人ボーナス¥2,200 | 代理店手数料の完全な単価表（トークン保護あり） |

**致命的な問題**: ゲスト向け料金（¥55,000）とホスト報酬（¥20,000+¥5,000=¥25,000）が両方公開されているため、**YumHomeStayのマージン（¥30,000/回）が計算できてしまう**。旅行代理店は手数料¥8,800を知った上で、ゲスト料金¥55,000も見えるため、「55,000円のうち8,800円しかもらえないのか」という勘繰りが発生します。

### 1.4 欠陥③：デモ動線が混乱

| 問題 | 詳細 |
|---|---|
| デモ入口が1つ | `/demo` に3種類が同居。ホスト向けLPから飛んでも料理教室・代理店のデモが見える |
| デモ出口が混在 | デモページ下部に3つのLP全てへのリンク |
| LP→デモの導線が遠い | LP → `/demo`（共通ページ）→ 選択 → ダッシュボード。2ステップ余分 |

## 2. 120点対応の設計

### 2.1 基本方針

**「各パートナーは自分の世界だけを見る」** を徹底します。

### 2.2 URL設計（変更後）

| パートナー | LP（公開） | ビジネスモデル（トークン保護） | デモ（独立入口） |
|---|---|---|---|
| ホスト | `/for-hosts` | `/business/host?token=xxx` | `/demo/host` |
| 料理教室 | `/for-cooking-schools` | `/business/cooking-school?token=xxx` | `/demo/cooking-school` |
| 旅行代理店 | `/for-agents` | `/business/agent?token=xxx` | `/demo/agent` |

旧 `/demo`（共通ページ）は削除し、各パートナー専用のデモ入口に置き換えます。

### 2.3 LP構成（各パートナー共通構造）

プロジェクトのナレッジに基づき、以下の構成で統一します。

| セクション | 内容 | 注意点 |
|---|---|---|
| ヒーローバナー + CTA | インパクト画像 + 「資料請求」「デモ体験」ボタン | 具体的な金額は出さない |
| キャッチコピー | お金以外の価値（国際交流・スキルアップ・地域貢献） | 感情に訴える |
| 背景にある問題点 | 各パートナーが抱える課題 | 共感を生む |
| サービスで得られる結果 | 収益性は「月収50,000円以上〜400,000円も可能」のみ | 個別単価は出さない |
| 3つの特徴 | 各パートナー固有のメリット | 差別化ポイント |
| 申込みのステップ | 資料請求→ビジネスモデル確認→シミュレーション→申込み | ファネルを明示 |
| キャンペーン特典 | 登録料無料キャンペーン（2026年8月まで） | 期間限定の訴求 |
| お客様の声 | 実際のパートナーの声（各パートナー種別のみ） | 他パートナーの声は混ぜない |
| FAQ | 各パートナー固有のFAQ | 他パートナーの情報は含めない |
| 資料請求フォーム + CTA | フォーム送信 → メールでビジネスモデルページへのリンク | ゲートキーパー |
| フッター | 自LP内のリンクのみ | 他パートナーLPへのリンクは排除 |

### 2.4 ビジネスファネル（情報開示の段階設計）

```
[LP（公開）]
  ├── 全体単価のみ表示（月収50,000円以上〜400,000円も可能）
  ├── お金以外の価値を強調
  ├── CTA: 「資料請求」「デモ体験」
  │
  ├──→ [資料請求フォーム送信]
  │     └── メールでトークン付きURLを送信
  │
  └──→ [デモ体験（/demo/host等）]
        └── 該当パートナーのデモのみ表示

[ビジネスモデルページ（トークン保護）] ← 資料請求後のみアクセス可能
  ├── ビジネスモデル説明
  ├── 収益単価の詳細
  ├── 収益シミュレーター
  ├── 画面スクリーンショット
  ├── 資料ダウンロード
  ├── デモログインリンク
  └── 本申込みフォーム
```

### 2.5 価格情報の非表示化ルール

| 情報 | LP（公開） | ビジネスモデル（トークン保護） |
|---|---|---|
| 全体収益レンジ（月収50,000円〜400,000円） | 表示OK | 表示OK |
| 個別単価（基本報酬¥20,000等） | **非表示** | 表示OK |
| 手数料体系（¥8,800等） | **非表示** | 表示OK |
| ゲスト向け料金（¥55,000等） | **非表示**（パートナーLPでは） | 不要 |
| シミュレーター | **非表示** | 表示OK |

### 2.6 ナビゲーション変更

**Navbar「パートナー」メニューの変更:**

現状（問題あり）:
- ホストファミリー向け → `/for-hosts`
- 料理教室向け → `/for-cooking-schools`
- 旅行代理店向け → `/for-agents`
- デモを試す → `/demo`（共通）

変更後:
- ホストファミリー向け → `/for-hosts`
- 料理教室向け → `/for-cooking-schools`
- 旅行代理店向け → `/for-agents`
- ~~デモを試す~~ → 削除（各LP内にデモリンクを設置）

**フッターの変更:**
- 「ホストになる」リンクのみ → 3パートナーそれぞれのLPリンクを追加

### 2.7 デモ動線の独立化

各パートナーのデモページ（`/demo/host`、`/demo/cooking-school`、`/demo/agent`）は以下の構造とします。

| 要素 | 内容 |
|---|---|
| ヘッダー | YumHomeStayロゴ + 「デモ環境」バッジ |
| 説明 | 該当パートナーのデモのみの説明 |
| デモログインボタン | 1つだけ（該当パートナーのみ） |
| 注意事項 | メール・決済は動作しない旨 |
| CTA | 「資料請求はこちら」→ 該当パートナーのLPのみ |

## 3. 実装計画

### Phase 1: LP刷新（ForHosts / ForCookingSchools / ForAgents）
- 価格・原価情報の削除
- お金以外の価値訴求コンテンツの追加
- 各LP内にデモリンク（`/demo/host`等）を設置
- 他パートナーへのリンクを完全排除

### Phase 2: デモページの独立化
- `/demo` を `/demo/host`、`/demo/cooking-school`、`/demo/agent` に分割
- 各デモページは該当パートナーのデモのみ表示
- 旧 `/demo` は適切なリダイレクト

### Phase 3: ナビゲーション・フッター修正
- Navbar「パートナー」メニューからデモリンクを削除
- フッターに3パートナーLPリンクを追加

### Phase 4: Pricing.tsxの修正
- パートナーLPからPricingページへのリンクを排除
- Pricingページ自体はゲスト向けとして残すが、パートナー向け導線からは切り離す
